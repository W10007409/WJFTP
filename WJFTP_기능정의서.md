## 1. 개요

WJFTP는 **SK-CDN(FTP)**과 **Naver Cloud(S3)** 두 가지 CDN 서비스에 파일을 업로드하고 관리하는 데스크탑 애플리케이션입니다.

| 항목 | 내용 |
|------|------|
| 앱 이름 | WJFTP - CDN Upload Manager |
| 기본 창 크기 | 1400 × 900 px |
| 최소 창 크기 | 1000 × 700 px |
| 지원 언어 | 한국어, English |
| 테마 | 다크 모드 (기본), 라이트 모드 |

### 화면 구성

```
┌──────────────────────────────────────────┐
│  접속 바 (CDN 선택 / 호스트 / 접속 버튼)     │
├──────────────────────────────────────────┤
│  탭 메뉴 (파일 탐색기 | 업로드 이력 | 설정)   │
├───────────────────┬──────────────────────┤
│  로컬 파일 탐색기    │  원격 파일 탐색기      │
│                   │                      │
│  [검색바]          │  [검색바]             │
│  [파일 목록]       │  [파일 목록]           │
│                   │                      │
│  [업로드→] [선택]   │  [새폴더] [이름변경]    │
│                   │  [삭제]               │
├───────────────────┴──────────────────────┤
│  전송 큐 (진행률 / 상태 / 퍼지 결과)         │
├──────────────────────────────────────────┤
│  로그 패널                                │
└──────────────────────────────────────────┘
```

## 2. 접속 관리

### 2.1 SK-CDN (FTP)

| 항목 | 내용 |
|------|------|
| 프로토콜 | FTP |
| 기본 호스트 | `wjth.ftp.scs.skcdn.co.kr` |
| 기본 포트 | 10021 |
| 입력 필드 | 호스트, 포트, 사용자명, 비밀번호 |

### 2.2 Naver Cloud (S3)

| 항목 | 내용 |
|------|------|
| 프로토콜 | S3 호환 API |
| 입력 필드 | 엔드포인트, Access Key, Secret Key |
| 리전 | `kr-standard` (고정) |
| 기본 엔드포인트 | `https://kr.object.ncloudstorage.com` |
| 버킷 | 접속 후 버킷 목록에서 선택 (입력 불필요) |

> 접속 시 버킷 목록이 자동으로 표시되며, 접근 권한이 있는 버킷과 없는 버킷이 구분됩니다.
> - 접근 가능: 일반 폴더 아이콘 (상단 정렬)
> - 접근 불가: 🔒 표시 + 반투명 (클릭 시 진입 차단)

### 2.3 공통 기능

- **자격증명 저장**: Electron safeStorage로 암호화 저장 (앱 재시작 시 자동 복원)
- **접속 상태 표시**: 녹색(접속됨) / 적색(미접속) 상태 표시등
- **원클릭 접속/해제**: 버튼 하나로 접속 토글

## 3. 로컬 파일 탐색기 (왼쪽 패널)

### 3.1 디렉토리 탐색

| 기능 | 설명 |
|------|------|
| 홈 버튼 | 사용자 홈 디렉토리로 이동 |
| 상위 폴더 | 한 단계 위로 이동 |
| 새로고침 | 현재 디렉토리 다시 로드 |
| 경로 직접 입력 | 경로 입력 후 Enter로 이동 |
| 마우스 뒤로가기 | 마우스 측면 뒤로가기 버튼으로 이전 디렉토리 이동 |
| 폴더 더블클릭 | 해당 폴더로 진입 |

### 3.2 파일 검색 및 필터

- **실시간 검색**: 파일명 입력 시 즉시 필터링
- **검색 결과 수**: 필터된 파일 수 / 전체 수 표시
- **대소문자 무시**: 검색 시 대소문자 구분 없음
- **클리어 버튼**: 검색어 한 번에 지우기

### 3.3 파일 목록 표시

| 열 | 내용 |
|----|------|
| 이름 | 파일/폴더명 + 확장자별 색상 아이콘 |
| 크기 | B / KB / MB / GB 자동 변환 |
| 수정일 | 로컬 시간대 기준 표시 |

- 폴더가 파일보다 먼저 정렬
- 숨김파일(`.`으로 시작)은 자동 제외
- 이름, 크기 기준 오름차순/내림차순 정렬 가능

### 3.4 파일 선택

| 조작 | 동작 |
|------|------|
| 클릭 | 마지막 클릭 위치 기억 (Shift 범위 기준점) |
| Ctrl + 클릭 | 개별 파일 선택/해제 토글 |
| Shift + 클릭 | 마지막 클릭 ~ 현재 클릭 범위 전체 선택 |
| Ctrl + Shift + 클릭 | 기존 선택에 범위 추가 |
| 체크박스 | Ant Design Table 기본 체크박스 선택 |

### 3.5 우클릭 컨텍스트 메뉴

로컬 파일 목록에서 마우스 오른쪽 버튼을 누르면 아래 메뉴가 표시됩니다:

| 메뉴 | 설명 |
|------|------|
| 업로드 | 선택한 파일을 원격 CDN에 업로드 |
| 폴더 열기 | 해당 폴더로 진입 (폴더만 활성) |
| 경로 복사 | 파일의 전체 경로를 클립보드에 복사 |
| 새로고침 | 현재 디렉토리 다시 로드 |

### 3.6 하단 액션 버튼

| 버튼 | 설명 |
|------|------|
| 업로드 → | 선택한 파일을 원격 서버로 업로드 (접속 시만 활성) |
| 파일 선택 | 시스템 파일 대화상자를 열어 업로드할 파일 선택 |

## 4. 원격 파일 탐색기 (오른쪽 패널)

### 4.1 기본 기능

- 로컬 탐색기와 동일한 탐색/검색/정렬 기능 제공
- CDN 유형 표시: 헤더에 `원격 파일 (SK-CDN)` 또는 `원격 파일 (NAVER)` 표시
- 접속 시 자동으로 루트(`/`) 디렉토리 로드

### 4.2 파일 관리

| 기능 | SK-CDN | Naver Cloud |
|------|--------|-------------|
| 새 폴더 | ✅ | ❌ (S3 제한) |
| 이름 변경 | ✅ | ✅ |
| 삭제 | ✅ (백업 후 삭제) | ✅ |

### 4.3 삭제 전 백업 (SK-CDN)

FTP에서 파일 삭제 시 자동으로 PC에 백업합니다:

- **백업 위치**: `{앱 데이터}/ftp-backup/{타임스탬프}/{원격경로}/파일명`
- **타임스탬프 형식**: `YYYY-MM-DDTHH-mm-ss`
- **파일**: 백업 후 삭제 (백업 실패 시 삭제하지 않음)
- **폴더**: 백업 없이 바로 삭제
- **삭제 확인**: 확인 대화상자에 백업 안내 메시지 표시

## 5. 파일 업로드

### 5.1 업로드 방법

| 방법 | 설명 |
|------|------|
| 로컬 탐색기 → 업로드 버튼 | 선택한 파일을 업로드 |
| 우클릭 → 업로드 | 컨텍스트 메뉴에서 업로드 |
| 파일 선택 버튼 | 시스템 파일 대화상자에서 파일 선택 후 업로드 |
| 드래그 & 드롭 | Windows 탐색기에서 파일을 앱 창에 끌어다 놓기 |

### 5.2 SK-CDN 파일명 변환

SK-CDN 업로드 시 경로와 파일명이 **자동으로 대문자**로 변환됩니다:

```
원본: /path/to/file.js
변환: /PATH/TO/FILE.JS
```

Naver Cloud는 원본 파일명을 그대로 유지합니다.

### 5.3 업로드 처리

- **병렬 업로드**: 선택한 모든 파일이 동시에 업로드
- **진행률 추적**: 파일별 실시간 업로드 진행률 표시
- **디렉토리 자동 생성**: 원격에 디렉토리가 없으면 자동 생성
- **폴더 구조 유지**: 폴더를 업로드하면 하위 구조 그대로 유지

### 5.4 자동 재시도

- **최대 3회** 자동 재시도
- 재시도 간 **2초 대기**
- 재시도 진행 상황 표시: `자동 재시도 (1/3)`
- 실패 시 수동 재시도 버튼 제공

## 6. CDN 캐시 퍼지

### 6.1 SK-CDN 자동 퍼지

SK-CDN에 파일 업로드 후 자동으로 CDN 캐시 무효화(퍼지)를 수행합니다.

| 항목 | 내용 |
|------|------|
| API 서버 | `https://cdn.openapi.wjthinkbig.com` |
| 엔드포인트 | `/auth/OpenApi.php` |
| 서비스 ID | `sdd` |
| 사용자 ID | `wjth@bookclubstudy` |

**도메인 매핑**

| FTP 경로 | CDN 도메인 |
|----------|-----------|
| `/WJTH_CACHE/...` | `https://cache.wjthinkbig.com/...` |
| `/WJTH_DOWNLOAD/...` | `https://down.wjthinkbig.com/...` |

> 위 경로 외의 파일은 퍼지를 스킵합니다 (에러 없음).

### 6.2 Naver Cloud Global Edge 자동 퍼지

Naver Cloud에 파일 업로드 후 Global Edge CDN 캐시를 자동으로 퍼지합니다.

| 항목 | 내용 |
|------|------|
| API 서버 | `https://edge.apigw.ntruss.com` |
| 엔드포인트 | `/api/v1/purge` |
| 인증 방식 | HMAC-SHA256 서명 (Access Key + Secret Key) |
| 설정 필요 | Access Key, Secret Key만 입력 (Profile ID, Edge ID는 자동 매핑) |

**버킷별 자동 매핑**: 각 버킷에 해당하는 Profile ID와 Edge ID가 프로그램에 내장되어 있어 별도 설정 없이 자동으로 퍼지됩니다. Edge ID가 2개인 버킷은 **양쪽 모두 퍼지**합니다.

> 매핑이 없는 버킷(ai-question-tool, anydevicepoc, userlog 등)은 퍼지 없이 업로드만 수행됩니다.

### 6.3 소스 코드 파싱 퍼지

HTML, JS, CSS 등의 소스 파일 업로드 시, 파일 내용을 분석하여 참조하는 리소스도 함께 퍼지합니다.

**대상 확장자**: `.html`, `.htm`, `.js`, `.jsx`, `.ts`, `.tsx`, `.css`, `.scss`, `.less`, `.mjs`

| 파일 유형 | 파싱 대상 |
|----------|----------|
| HTML | `src=`, `href=`, `action=`, `data-src=`, `poster=` 속성 |
| CSS | `url(...)` 선언 |
| JS/TS | `import "..."`, `require("...")`, `import("...")` |

**제외 대상**: `data:` URL, `#` 앵커, `mailto:`, `javascript:`, 외부 도메인 URL

### 6.5 퍼지 오류 처리

- 개별 URL 실패 시 다른 URL은 계속 퍼지
- 실패한 URL별 오류 메시지 표시
- 전송 큐에서 퍼지 실패 태그에 마우스 올리면 상세 오류 확인 가능

## 7. 전송 큐

### 7.1 큐 표시

- **접기/펼치기**: 헤더 클릭으로 전송 큐 토글
- **전체 진행률**: 완료된 파일 수 / 전체 파일 수
- **개별 항목**: 원본 파일명 → 업로드 파일명, 진행률, 상태

### 7.2 전송 상태

| 상태 | 색상 | 설명 |
|------|------|------|
| 대기 | 회색 | 큐에 추가됨, 아직 시작 안 함 |
| 업로드 중 | 파랑 | 파일 전송 중 |
| 퍼지 중 | 주황 | CDN 캐시 무효화 진행 중 |
| 완료 | 녹색 | 업로드 및 퍼지 성공 |
| 실패 | 빨강 | 재시도 소진 후 최종 실패 |
| 재시도 중 | 주황 | 자동 재시도 진행 중 |

### 7.3 큐 관리

| 버튼 | 설명 |
|------|------|
| 완료 항목 제거 | 성공한 항목을 큐에서 일괄 삭제 |
| 일괄 퍼지 | 완료된 항목의 퍼지를 수동으로 재실행 |
| 개별 삭제 | 항목별 삭제 버튼 |
| 수동 재시도 | 실패 항목에 표시되는 재시도 버튼 |

## 8. 업로드 이력

### 8.1 통계

| 항목 | 설명 |
|------|------|
| 전체 업로드 | 총 업로드 시도 횟수 |
| 성공 | 성공한 업로드 수 |
| 실패 | 실패한 업로드 수 |
| 총 용량 | 업로드한 파일의 총 크기 |

### 8.2 필터링 및 검색

- **텍스트 검색**: 파일명, 원격 경로로 검색
- **CDN 필터**: SK-CDN / Naver Cloud / 전체
- **상태 필터**: 성공 / 실패 / 전체
- **페이지네이션**: 50건 단위 페이지 분할

### 8.3 상세 보기

이력 항목 클릭 시 상세 정보를 확인할 수 있습니다:

- CDN 유형
- 원본 파일명 / 업로드 파일명
- 원격 경로
- 파일 크기
- 상태 및 오류 메시지
- 퍼지 URL 목록

### 8.4 데이터 관리

- **영구 저장**: SQLite(sql.js) 기반 로컬 데이터베이스
- **저장 위치**: `{앱 데이터}/wjftp_history.db`
- **전체 삭제**: 확인 후 모든 이력 삭제 가능

## 9. 설정

### 9.1 CDN 설정

- **SK-CDN**: 호스트, 포트, 사용자명, 비밀번호 저장
- **Naver Cloud**: 엔드포인트, Access Key, Secret Key 저장
- **Global Edge 퍼지**: 버킷별 Profile ID / Edge ID 자동 매핑 (수동 입력 불필요)
- 저장 시 비밀번호/키는 암호화 처리

### 9.2 백업

- **백업 폴더 열기** 버튼으로 Windows 탐색기에서 백업 디렉토리 바로 접근
- 백업 위치: `%APPDATA%\WJFTP\ftp-backup\`

### 9.3 외관 설정

| 설정 | 옵션 |
|------|------|
| 테마 | 다크 (기본) / 라이트 |
| 언어 | 한국어 (기본) / English |

- 변경 즉시 적용 (재시작 불필요)
- 설정값 영구 저장

### 10.2 마우스

| 조작 | 동작 |
|------|------|
| 더블클릭 | 폴더 진입 |
| 우클릭 (로컬) | 컨텍스트 메뉴 표시 |
| 마우스 뒤로가기 버튼 | 이전 디렉토리로 이동 |
| 드래그 & 드롭 | Windows 탐색기에서 파일을 끌어다 놓으면 업로드 |

## 11. 보안

### 11.1 자격증명 보호

### 11.2 앱 보안

- **Context Isolation**: 렌더러 프로세스와 메인 프로세스 격리
- **Node Integration 비활성화**: 렌더러에서 Node.js API 직접 접근 차단
- **Preload 스크립트**: 필요한 API만 선택적으로 노출

## Naver Cloud 접속 키 발급 가이드

WJFTP에서 Naver Cloud에 접속하려면 **본안(메인) 계정 또는 서브계정의 Access Key / Secret Key**가 필요합니다.

### Access Key / Secret Key 발급 방법

1. [NCP 콘솔](https://console.ncloud.com)에 로그인
2. 우측 상단 프로필 아이콘 클릭 → **마이페이지** 이동
3. **보안 관리 > 접근 관리** 메뉴 선택
4. **API 인증키 관리** 에서 새 Access Key 생성
5. **Access Key ID**와 **Secret Key**를 복사하여 WJFTP 설정에 입력

> Secret Key는 생성 시 한 번만 표시됩니다. 반드시 안전한 곳에 복사해 두세요.

### 서브계정 사용 시 추가 설정

서브계정으로 접속하는 경우 아래 권한이 추가로 필요합니다:

| 설정 항목 | 경로 | 설명 |
|----------|------|------|
| API Gateway 접근 | Sub Account > 해당 계정 > **API Gateway 접근** 체크 | API 호출 권한 활성화 |
| Object Storage 권한 | Sub Account > Policies > 정책 생성 | 버킷별 접근 권한 설정 |
| Global Edge 권한 | Sub Account > 해당 계정 > 권한 > **Global Edge** 추가 | 퍼지 API 호출 권한 |

#### 버킷별 접근 제한 (서브계정)

1. NCP 콘솔 → **Sub Account** → **Policies** → **User Created** 탭
2. 정책 생성 클릭
3. Resource-based 조건 추가:
   - `ncp:resource:oss:bucket:{버킷이름}` 형태로 허용할 버킷 지정
4. 해당 정책을 서브계정에 연결

> WJFTP에서 접속하면 접근 가능한 버킷은 일반 아이콘으로, 접근 불가 버킷은 🔒 아이콘으로 구분 표시됩니다.
