# WJFTP - CDN 업로드 관리 프로그램

## 개요

WJFTP는 **SK-CDN(FTP)**과 **Naver Cloud Object Storage(S3)** 두 가지 CDN에 파일을 업로드하고, 업로드 후 자동으로 캐시 퍼지까지 수행하는 Windows 데스크톱 프로그램입니다.

---

## 설치

1. `WJFTP Setup 1.0.0.exe` 실행
2. 설치 경로 선택
3. **바탕화면 바로가기 생성** 체크박스 선택 가능
4. 설치 완료 후 바로 실행

---

## 주요 기능

### 1. 듀얼 CDN 지원

| 항목 | SK-CDN | Naver Cloud |
|------|--------|-------------|
| 프로토콜 | FTP | S3 (Object Storage) |
| 인증 | 호스트/포트/계정/비밀번호 | Endpoint/Access Key/Secret Key |
| 퍼지 | SK-CDN OpenAPI 자동 퍼지 | Global Edge 자동 퍼지 |
| 파일명 | 자동 대문자 변환 | 원본 유지 |

### 2. 접속

상단 연결 바에서 CDN 타입을 선택하고 인증 정보를 입력한 뒤 **접속** 버튼을 클릭합니다.

- **SK-CDN**: 호스트, 포트, 사용자, 비밀번호 입력
- **Naver Cloud**: Endpoint, Access Key, Secret Key 입력 (버킷 입력 불필요)

> 한 번 접속에 성공하면 인증 정보가 자동 저장되어 다음 접속 시 다시 입력할 필요 없습니다.

### 3. 파일 탐색기 (좌: 로컬 / 우: 원격)

#### 로컬 파일 탐색기 (왼쪽)
- PC의 파일/폴더를 탐색
- 드라이브 목록 표시
- 파일명 검색 기능
- 우클릭 메뉴: 업로드, 폴더 열기, 경로 복사, 새 폴더, 이름 변경, 삭제

#### 원격 파일 탐색기 (오른쪽)
- **SK-CDN**: FTP 서버의 디렉토리/파일 탐색
- **Naver Cloud**: 접속 시 **버킷 목록**이 최상위에 표시됨
  - 접근 가능한 버킷: 일반 폴더 아이콘
  - 접근 불가 버킷: 🔒 표시 + 반투명 (클릭 시 진입 차단)
  - 버킷 선택 후 하위 폴더/파일 탐색
- 우클릭 메뉴: 폴더 열기, 경로 복사, 새 폴더, 이름 변경, 삭제

### 4. 파일 업로드

#### 업로드 방법 (3가지)
1. **드래그 앤 드롭**: 로컬 파일을 프로그램 창에 드래그
2. **우클릭 → 업로드**: 로컬 파일 선택 후 우클릭 메뉴에서 업로드
3. **다중 선택 업로드**: Ctrl/Shift 클릭으로 여러 파일 선택 후 업로드

#### 업로드 특징
- 폴더 단위 업로드 지원 (하위 구조 유지)
- SK-CDN: 파일명/경로 자동 대문자 변환
- Naver Cloud: 원본 파일명 유지
- 전송 큐에서 진행 상태 실시간 확인
- 업로드 실패 시 자동 재시도 (최대 3회)

### 5. 자동 캐시 퍼지

파일 업로드 완료 후 CDN 캐시를 자동으로 퍼지합니다.

#### SK-CDN 퍼지
- 업로드 완료 후 일괄 퍼지 자동 실행
- `/WJTH_CACHE` → `cache.wjthinkbig.com` 매핑
- `/WJTH_DOWNLOAD` → `down.wjthinkbig.com` 매핑
- 매핑되지 않는 경로는 자동 스킵 (에러 없음)
- HTML/CSS/JS 파일 업로드 시 참조된 리소스도 함께 퍼지

#### Naver Cloud Global Edge 퍼지
- 설정에서 **Profile ID**와 **Edge ID** 입력 필요
- 업로드 완료 후 개별 파일 자동 퍼지
- 설정하지 않으면 퍼지 없이 업로드만 수행

### 6. 전송 큐

화면 하단에 전송 진행 상황을 실시간으로 표시합니다.

| 상태 | 설명 |
|------|------|
| 대기 | 업로드 순서 대기 중 |
| 업로드 중 | 파일 전송 진행 중 |
| 퍼지 중 | CDN 캐시 퍼지 진행 중 |
| 완료 | 업로드 + 퍼지 완료 |
| 실패 | 업로드 또는 퍼지 실패 |
| 재시도 중 | 자동 재시도 진행 중 |

- **전체 취소**: 대기 중인 모든 전송 취소
- **완료 항목 제거**: 완료된 항목을 큐에서 정리

### 7. 원격 파일 삭제 (SK-CDN)

SK-CDN에서 파일 삭제 시 **자동 백업** 후 삭제됩니다.

- 삭제 전 파일을 로컬 PC에 자동 다운로드
- 백업 위치: `%APPDATA%\WJFTP\ftp-backup\`
- 설정 페이지에서 **백업 폴더 열기** 버튼으로 바로 접근 가능

### 8. 업로드 이력

모든 업로드 기록이 자동으로 저장됩니다.

- CDN 타입별 필터
- 상태별 필터 (성공/실패)
- 날짜 범위 검색
- 파일명 검색
- 통계: 전체 업로드 수, 성공/실패 수, 총 용량
- 퍼지 URL 목록 확인

### 9. 로그 패널

화면 최하단에 실시간 작업 로그가 표시됩니다.

- 접속/해제 상태
- 업로드 성공/실패
- 퍼지 결과
- 에러 메시지

## 설정

설정 탭에서 다음 항목을 관리합니다.

### SK-CDN (FTP) 설정
- 호스트 (기본: `wjth.ftp.scs.skcdn.co.kr`)
- 포트 (기본: `10021`)
- 사용자 / 비밀번호

### Naver Cloud Object Storage 설정
- Endpoint (기본: `https://kr.object.ncloudstorage.com`)
- Access Key / Secret Key
- **Global Edge 퍼지 설정**
  - Profile ID
  - Edge ID

### 백업
- 백업 폴더 열기 버튼

### 외관
- **테마**: 다크 / 라이트 (우클릭 메뉴 포함 전체 적용)
- **언어**: 한국어 / English

## Naver Cloud 서브계정 설정 가이드

### Access Key 발급
1. NCP 콘솔 → Sub Account → 해당 서브계정 선택
2. **"API Gateway 접근"** 체크박스 활성화 (필수)
3. API Key 탭 → API Key 생성
4. Access Key와 Secret Key 복사

### 버킷 접근 제어 (서브계정별 버킷 제한)
1. NCP 콘솔 → Sub Account → Policies
2. **User Created** 탭 → 정책 생성
3. Resource-based 조건 추가:
   - `ncp:resource:oss:bucket:{버킷이름}` 형태로 허용할 버킷 지정
4. 해당 정책을 서브계정에 연결

### Global Edge 퍼지 권한
- 서브계정에 **Global Edge** 서비스 권한 추가 필요
- NCP 콘솔 → Sub Account → 해당 계정 → 권한 → Global Edge 추가

## 참고 사항

- 인증 정보(비밀번호, Secret Key)는 Windows의 `safeStorage`를 통해 암호화 저장됩니다.
- SK-CDN FTP는 동시에 하나의 작업만 가능하므로 파일이 순차적으로 업로드됩니다.
- Naver Cloud는 리전이 `kr-standard`로 고정되어 있습니다.
- 프로그램 데이터 저장 위치: `%APPDATA%\WJFTP\`
