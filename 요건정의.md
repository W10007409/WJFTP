## 1. 프로젝트 개요

### 1.1 목적
SK-CDN과 Naver Cloud Object Storage 2개 CDN에 파일을 업로드하고 관리하는 **데스크톱 애플리케이션** 개발.
FileZilla 스타일의 직관적인 UI/UX를 제공하되, CDN 퍼지 자동화 등 더 편리한 구조로 설계한다.

### 1.2 지원 CDN
| CDN | 업로드 방식 | 퍼지 | 비고 |
|-----|------------|------|------|
| SK-CDN | FTP (일반, 포트 10021) | 필수 (프록시 API) | 파일명/경로 대문자 변환 필요 |
| Naver Cloud Object Storage | S3 호환 API | 퍼지 API 존재 시 적용 | 파일명 원본 유지 |

### 1.3 대상 사용자
- **팀 전체** (5명 이상)
- 개발자 및 비개발자 혼합 사용

### 1.4 일정
- **급하지 않음** - 품질 우선 개발
- 단계별 개발: SK-CDN 먼저 → Naver Cloud 추가

## 2. SK-CDN 상세

### 2.1 FTP 접속 정보
| 항목 | 값 |
|------|-----|
| 호스트 | `wjth.ftp.scs.skcdn.co.kr` |
| 포트 | `10021` |
| 프로토콜 | 일반 FTP (비암호화) |
| 인증 | 사용자 ID/PW (앱 내 암호화 저장) |

### 2.2 업로드 규칙
- **대문자 변환 범위**: 전체 경로 + 파일명 + 확장자를 모두 **대문자로 변환**하여 업로드
  - 예: `contents/style.css` → `CONTENTS/STYLE.CSS`
  - 예: `images/logo.png` → `IMAGES/LOGO.PNG`
- **경로 규칙**: 자유 경로 (고정 prefix 없음, FTP 접속 후 원하는 위치에 업로드)
- **폴더 업로드**: 폴더 구조를 유지하며 업로드

### 2.3 퍼지 (Cache Purge)

#### 2.3.1 퍼지 프록시 API
- **프록시 서버**: `https://any-web-api.wjthinkbig.com`
- **엔드포인트**: `/cdn-api/auth/OpenApi.php`
- **방식**: GET 요청
- **파라미터**:
  | 파라미터 | 값 | 설명 |
  |---------|-----|------|
  | `a` | `purge` | 퍼지 액션 |
  | `file` | `base64(CDN URL)` | 퍼지 대상 URL (Base64 인코딩) |
  | `ttl` | `0` | TTL |
  | `service` | `sdd` | 서비스 코드 (고정) |
  | `userID` | `wjth@bookclubstudy` | 사용자 ID (고정) |

#### 2.3.2 CDN 도메인 매핑 (퍼지 URL 생성 규칙)
| FTP 업로드 경로 prefix | CDN 다운로드 도메인 |
|------------------------|---------------------|
| `/WJTH_CACHE/...` | `https://cache.wjthinkbig.com/...` |
| `/WJTH_DOWNLOAD/...` | `https://down.wjthinkbig.com/...` |

- 이 2가지 매핑은 **고정**이며 변경 없음

#### 2.3.3 퍼지 타이밍
1. **파일별 즉시 퍼지**: 각 파일 업로드 완료 직후 즉시 퍼지 요청
2. **일괄 퍼지**: 모든 업로드 완료 후 전체 파일에 대해 한 번 더 일괄 퍼지 실행

#### 2.3.4 HTML/JS/CSS 파일 특수 퍼지 처리
- **문제**: SK-CDN은 파일명/경로를 대문자로 변환하여 업로드하지만, HTML/JS/CSS 파일 내부의 경로 참조는 원본 대소문자를 사용함
- **해결**: HTML, JS, CSS 파일은 소스코드를 파싱하여 내부 경로를 추출하고, 추출된 경로 그대로(원본 대소문자) 퍼지 요청
- **파싱 대상**: `src`, `href`, `url()`, `import` 등 모든 경로 참조 패턴
- **퍼지 범위**: 파일 자체 경로(대문자) + 내부 참조 경로(원본 대소문자) 모두 퍼지

## 3. Naver Cloud Object Storage 상세

### 3.1 접속 정보
- **방식**: AWS S3 호환 API
- **접속 설정**: 앱 설정 화면에서 입력
  - Endpoint URL
  - Bucket 이름
  - Region
  - Access Key
  - Secret Key
- 접속 정보는 앱 내 암호화 저장

### 3.2 업로드 규칙
- **파일명**: 원본 유지 (대문자 변환 없음)
- **특수 규칙 없음**
- 폴더 구조 유지하며 업로드

### 3.3 파일 관리
- SK-CDN과 동일한 기능 제공: 탐색, 업로드, 삭제, 이름 변경

### 3.4 퍼지
- Naver Cloud CDN 퍼지 API가 존재하면 적용
- 개발 시 API 존재 여부 조사 후 결정

### 4.3 주요 기능 영역

#### 4.3.1 상단: 접속 설정
- CDN 선택 드롭다운 (SK-CDN / Naver Cloud)
- 접속 정보 입력 (CDN별 다른 필드)
- 접속/해제 버튼
- 접속 상태 표시

#### 4.3.2 좌측: 로컬 파일 탐색기
- 로컬 디렉토리 트리 탐색
- 파일/폴더 선택 (다중 선택 지원)
- 경로 직접 입력 가능

#### 4.3.3 우측: CDN 원격 파일 탐색기
- FTP/S3 서버의 디렉토리/파일 목록 표시
- 파일 관리: 삭제, 이름 변경, 새 폴더 생성
- 경로 직접 입력 가능

#### 4.3.4 하단: 전송 큐 및 상태
- 개별 파일 진행률 (프로그레스 바)
- 전체 진행률 (N/M 완료)
- 파일별 상세 정보:
  - 원본 파일명 → 변환된 파일명 (SK-CDN 대문자)
  - 파일 크기
  - 전송 속도
  - 남은 시간 예측
  - 상태: 대기/업로드중/퍼지중/완료/실패
  - FTP 응답 코드
  - 퍼지 결과
  - 대문자 변환 전/후 파일명
- 실패 건 재시도 버튼
- 일괄 퍼지 버튼
- 실시간 로그 표시

### 4.4 업로드 방식 (모두 지원)

3. **파일/폴더 선택 대화상자**: 시스템 파일 선택 다이얼로그

### 4.6 다국어

### 6.1 FTP 파일 관리 (SK-CDN)
| 기능 | 설명 |
|------|------|
| 접속/해제 | FTP 접속 및 연결 해제 |

| 이름 변경 | 원격 파일/폴더 이름 변경 |
| 새 폴더 생성 | 원격 디렉토리 생성 |

### 6.2 S3 파일 관리 (Naver Cloud)
| 기능 | 설명 |
|------|------|
| 접속/해제 | S3 API 인증 및 연결 |
| 버킷/디렉토리 탐색 | Object 목록 탐색 |
| 파일 업로드 | 원본 파일명 그대로 업로드 |
| 파일 삭제 | Object 삭제 |
| 이름 변경 | Object 복사 후 원본 삭제 (S3 방식) |

### 6.3 퍼지 처리 흐름

#### SK-CDN 일반 파일 퍼지:
```
1. 파일 업로드 완료
2. FTP 업로드 경로에서 CDN 도메인 매핑
   - /WJTH_CACHE/path/FILE.JPG → https://cache.wjthinkbig.com/path/FILE.JPG
   - /WJTH_DOWNLOAD/path/FILE.JPG → https://down.wjthinkbig.com/path/FILE.JPG
3. 퍼지 API 호출 (base64 인코딩된 URL)
```

#### SK-CDN HTML/JS/CSS 파일 퍼지:
```
1. 파일 업로드 완료
2. 파일 소스코드 파싱 → 내부 경로 추출
   - <script src="/js/app.js">  → /js/app.js
   - <link href="/css/style.css"> → /css/style.css
   - url('/images/bg.png')  → /images/bg.png
   - import './module.js'   → ./module.js (상대경로 → 절대경로 변환)
3. 파일 자체 CDN URL + 내부 참조 경로들을 모두 퍼지
   - 파일 자체: 대문자 변환된 경로로 퍼지
   - 내부 참조: 소스코드에 적힌 원본 대소문자 그대로 퍼지
```

### 6.4 재시도 정책
- **자동 재시도**: 업로드 실패 시 최대 **3회** 자동 재시도
- **수동 재시도**: 자동 재시도 실패 후 UI에서 수동 재시도 가능

### 6.5 동시 전송
- **제한 없음**: 선택한 모든 파일을 동시 업로드
  - (실제 구현 시 FTP 연결 수 제한 등 기술적 한계 내에서 최대한 병렬 처리)

### 6.6 인증 정보 관리
- **SK-CDN**: FTP ID/PW를 Electron `safeStorage`로 암호화하여 로컬 저장
- **Naver Cloud**: Endpoint, Bucket, Region, Access Key, Secret Key를 암호화 저장
- 앱 시작 시 저장된 인증 정보 자동 복원

## 7. 업로드 이력 관리

### 7.1 저장소
- **SQLite 로컬 DB** (`wjftp_history.db`)

### 7.2 이력 테이블 스키마 (예상)
```sql
CREATE TABLE upload_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    cdn_type TEXT NOT NULL,           -- 'SK-CDN' | 'NAVER'
    local_path TEXT NOT NULL,         -- 로컬 원본 경로
    remote_path TEXT NOT NULL,        -- CDN 업로드 경로
    original_filename TEXT NOT NULL,  -- 원본 파일명
    uploaded_filename TEXT NOT NULL,  -- 업로드된 파일명 (대문자 변환 후)
    file_size INTEGER,               -- 파일 크기 (bytes)
    status TEXT NOT NULL,             -- 'success' | 'failed' | 'purged'
    purge_status TEXT,                -- 'success' | 'failed' | 'pending' | NULL
    purge_urls TEXT,                  -- 퍼지한 URL 목록 (JSON)
    error_message TEXT,               -- 실패 시 에러 메시지
    retry_count INTEGER DEFAULT 0,    -- 재시도 횟수
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    completed_at DATETIME
);
```

### 7.3 이력 UI
- 이력 검색/필터링 (날짜, CDN, 상태별)
- 이력 상세 보기 (퍼지 URL 목록 포함)

## 8. 트레이드오프 및 결정사항

### 8.1 결정된 사항
| 결정 | 선택 | 이유 |
|------|------|------|
| 앱 형태 | Electron 데스크톱 앱 | 로컬 파일 접근, FTP 직접 연결, 드래그&드롭 지원 용이 |
| UI 레이아웃 | 좌우 분할 (FileZilla 스타일) | FTP 클라이언트에 익숙한 사용자들이 직관적으로 사용 |
| FTP 인증 저장 | 앱 내 암호화 저장 | 팀 다수 사용, 매번 입력 불편 |
| Naver Cloud 연동 | S3 호환 API | 표준 프로토콜, 라이브러리 풍부 |
| 퍼지 타이밍 | 즉시 + 일괄 | 파일별 즉시 퍼지 후, 완료 시 일괄 퍼지 한 번 더 |
| 대문자 변환 범위 | 전체 경로 + 파일명 | SK-CDN 요구사항 |
| 재시도 정책 | 자동 3회 | 네트워크 일시 장애 대응 |
| 이력 저장 | SQLite 영구 저장 | 추후 이력 조회/감사 가능 |
| 동시 전송 | 제한 없음 | 사용자 요청, FTP 연결 수 내에서 최대 병렬 |
| 파일 관리 | 업로드 + 삭제 + 이름변경 | 완전한 파일 관리 도구 |
| 원격 탐색 | 필수 (양쪽 CDN 모두) | FileZilla 스타일 필수 기능 |
| 배포 | .exe 설치 파일 | 팀 배포 용이 |
| 테마 | 다크/라이트 선택 가능 | 사용자 선호 반영 |
| 언어 | 한국어 + 영어 | 다국어 지원 |

### 8.2 기술적 우려사항
| 우려 | 대응 방안 |
|------|----------|
| 일반 FTP 보안 (비암호화) | 내부망 사용 전제, ID/PW 로컬 암호화 저장으로 최소한의 보호 |
| 대용량 파일 (GB급) 업로드 | FTP 스트리밍 업로드, 진행률 표시, 연결 끊김 시 재시도 |
| HTML/JS/CSS 경로 파싱 정확도 | 정규식 + AST 파싱 병용, 동적 경로는 감지 불가 안내 |
| FTP 동시 연결 수 제한 | 서버 제한에 맞춰 연결 풀 관리 |
| Naver Cloud 퍼지 API 미확인 | 개발 시 API 조사 후 결정 |

### 8.3 향후 고려 가능 사항 (현재 범위 외)
- 자동 업데이트 기능
- 예약 업로드 (스케줄링)
- 다운로드 기능
- CLI 모드
- 폴더 동기화 기능

## 9. 용어 정의

| 용어 | 설명 |
|------|------|
| SK-CDN | SK브로드밴드 CDN 서비스 |
| Naver Cloud Object Storage | 네이버 클라우드 플랫폼의 오브젝트 스토리지 서비스 (S3 호환) |
| 퍼지 (Purge) | CDN 캐시를 강제로 갱신하는 작업. 파일 업로드 후 캐시된 이전 버전을 제거 |
| FTP | File Transfer Protocol. 파일 전송 프로토콜 |
| FTPS | FTP over TLS/SSL. 암호화된 FTP |
| basic-ftp | Node.js FTP 클라이언트 라이브러리 |
| electron-store | Electron 앱의 로컬 데이터 저장 라이브러리 |
| safeStorage | Electron의 OS 수준 암호화 API |
